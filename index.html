📄 Page 1: First Concepts — Cyber, Security, CIA, DoS, DDoS

❓ What is required of an attack to be regarded as a cyber attack?

✅ 1. The target relies on an electronic information system.
💡 A cyber attack simply requires that the target is an electronic information system. Nation state involvement, substantial loss and large groups are common but NOT necessary.

❓ Denial of service is a term...

✅ 3. That refers to a service not being able to operate because of malicious requests.
💡 DoS means malicious traffic overwhelms the service. Option 2 is too broad. Option 1 is too specific. Option 4 describes admin action not an attack.

❓ What makes a DoS a distributed DoS?

✅ 4. The attacking traffic comes to the server from several computers simultaneously.
💡 Distributed means attack traffic originates FROM many sources simultaneously. Option 1 confuses many hackers with many computers.

❓ The term attack vector...

✅ 3. Refers to the method of an attack — one that happened or is possible.
💡 Attack vector is the method or path used in an attack. Not the attack itself, not the attacker's view of defenses, not a list of vulnerabilities.

❓ Confidentiality is one of the three basic CIA goals...

✅ 3. But it is usually not sufficient without integrity and availability.
💡 All three CIA goals work together. Confidentiality alone is insufficient.

📄 Page 2: Some Victims and Attackers — Script Kiddies, Identity Theft, Social Engineering

❓ Script kiddie — what is typical from a defender's perspective?

✅ 1. Essentially similar scripts to several other attackers.
💡 Script kiddies use the same ready-made tools so attacks are recognizable and blockable.

❓ Attackers called script kiddies are characterized by...

✅ 1. Using scripts written by others.
💡 Script kiddies use attack tools written by more skilled hackers without understanding how they work.

❓ Identity theft...

✅ 1. Happens when someone maliciously uses identifying data of another person.
💡 Identity theft requires malicious intent. Can happen without financial loss.

❓ How many user-spread entities are NOT direct risks to computing: spam, hoax, ransomware, meme virus, Nigerian letter, troll?

✅ 1. 4
💡 Ransomware is a direct risk. Hoax, meme virus, Nigerian letter and troll are NOT direct computing risks. That gives 4 entities.

❓ Which description gives the best coverage for social engineering?

✅ 4. Exercising the art of influencing people to act against their security policy.
💡 Social engineering is the broad concept of manipulating people psychologically. Options 1, 2 and 3 are specific techniques not the full concept.

❓ Successful impersonation means that...

✅ 4. Somebody/something has been cheated to act as if in contact with someone else than the attacker.
💡 Impersonation means fooling someone into thinking they deal with a trusted party. About deception not identity assignment.

📄 Page 3: Not Only Breaking Ciphers — Eavesdropper, Dictionary Attack, Backdoor, Ransomware, Passive Attack

❓ Attacker who attempts to know conversation contents without being a legitimate party?

✅ 4. Eavesdropper.
💡 Eavesdropper listens without being a legitimate party. Honest but curious is a legitimate party trying to learn more than they should.

❓ What is a dictionary attack?

✅ 2. A sophisticated brute-force attack against passwords, taking into account that they were invented by users of natural languages.
💡 Dictionary attack tries common words instead of random combinations. Option 3 describes rainbow tables.

❓ Backdoor?

✅ 4. Is an intentionally made or maliciously installed method of bypassing normal authentication or encryption.
💡 A backdoor bypasses normal security controls. Option 2 describes Trojan horse payload.

❓ Ransomware is?

✅ 2. A program that makes your data inaccessible to you and asks for money to restore access.
💡 Ransomware encrypts files and demands payment. Data is locked not destroyed.

❓ How can an attack be passive?

✅ 3. A passive attack does not interfere with the integrity or availability of the victim's computing.
💡 A passive attack only observes data without modifying anything. Eavesdropping is the classic example.

📄 Page 4: Not Only Viruses — Malware, Trojan Horse, Macro Virus, Exposure, Buffer Overflow, Zero-day

❓ Malware that does not replicate, pretends to be legitimate but does something else?

✅ 4. A Trojan horse.
💡 Trojan horse disguises itself as legitimate but performs malicious actions. Does NOT replicate unlike virus or worm.

❓ Common feature of all malicious programs?

✅ 1. They are capable of similar actions as other programs in their running environment.
💡 All malware runs in the same environment as legitimate software and can do anything that environment allows.

❓ A macro virus is malware that...

✅ 4. Runs on many different platforms because it is interpreted by its host program.
💡 Macro viruses run in host programs like Word or Excel. Cross-platform ability is the key characteristic.

❓ Term for a vulnerability meaning something coming out of covering?

✅ 3. Exposure.
💡 Exposure is the security term for a vulnerability that reveals something hidden. Disclosure means revealing intentionally.

❓ Attack causing data written outside allowed memory area?

✅ 3. Buffer overflow.
💡 Buffer overflow occurs when a program writes more data than a buffer can hold, spilling into adjacent memory.

❓ The term zero-day applies to...

✅ 3. A vulnerability in software not yet exploited but found and kept secret by a malicious party.
💡 Zero-day vulnerability is unknown to the vendor — they have had zero days to fix it.

📄 Page 5: Networked Robots and Men — Botnet, Man in the Middle

❓ Which fits most poorly to the concept of a bot network?

✅ 1. Users of machines on the bot network have agreed to work with the network administrator.
💡 A botnet is built from compromised machines whose owners do NOT know. If users agreed it would be a legitimate distributed network not a botnet.

❓ Man in the middle is an attack type where...

✅ 2. The attacker or his process relays modified messages between two unknowing communication parties.
💡 In MITM the attacker positions between two parties, intercepts and can modify messages. Option 1 describes a rootkit. Option 3 describes a cryptographic attack.

❓ A botnet is...

✅ 2. A group of dispersed, compromised machines controlled remotely for illicit purposes.
💡 A botnet is compromised computers controlled by an attacker without owners' knowledge. Used for DDoS, spam, credential theft.

📄 Page 6: Risk and Three A-Properties — Assurance, Authentication, Availability, Risk

❓ Assurance is just part of information security, namely...

✅ 3. Evidence that security mechanisms are efficient.
💡 Assurance is confidence or evidence that security measures actually work. Not a synonym for authentication or accountability.

❓ Authentication is the...

✅ 4. Process of verifying an identity.
💡 Authentication verifies that someone is who they claim to be. Option 1 describes identification. Option 2 describes authorization. Option 3 describes an access management decision.

❓ Availability protects from some but not all — how many are excluded from: DoS, fire, flood, unauthorized transaction, unreadable backup tape?

✅ 4. 3
💡 Fire, flood and unauthorized transaction are excluded. Fire and flood are physical/disaster threats. Unauthorized transaction is an integrity issue. So 3 are excluded.

❓ Which coupling best defines risk?

✅ 1. Threat and vulnerability.
💡 Risk = Threat x Vulnerability. Without a vulnerability a threat cannot succeed. Without a threat a vulnerability does not matter.

📄 Page 7: Policies and Models Do Not Protect — Access Control, Accountability, Non-repudiation, Defence in Depth

❓ Access control means...

✅ 3. Measures, usually automatic, taken to decide whether access to a resource must be granted or denied, based on a policy.
💡 Access control is the enforcement mechanism. Option 2 describes policy. Option 4 is too broad. Option 1 is too narrow.

❓ Accountability is close to Authenticity combined with which CIA goal?

✅ 1. I (Integrity)
💡 Accountability requires authenticity combined with integrity of audit records. Without integrity of logs, accountability is meaningless.

❓ How many of 7 terms mean roughly the same as Security control: mechanism, model, policy, service, protection, countermeasure, safeguard?

✅ 4. 5
💡 Mechanism=yes, model=NO, policy=NO, service=yes, protection=yes, countermeasure=yes, safeguard=yes. That gives 5.

❓ The strategy of forming layers of protection around an asset is known as...

✅ 2. Defence-in-depth.
💡 Defence-in-depth uses multiple layers so if one fails others still protect. Other options are not standard security terms.

❓ Non-repudiation as a security objective means that...

✅ 4. The person to whom the information relates has no grounds for claiming that there is no connection.
💡 Non-repudiation means a party cannot deny having sent or received a message. Evidence is so strong that denial is impossible.

❓ Non-repudiation is...

✅ 1. A special security goal.
💡 Non-repudiation is a security goal ensuring parties cannot deny their actions. It sits alongside the CIA triad as a separate special goal.

📄 Page 8: Credentials — 2FA, Credentials, Data Ownership, Shared Secret

❓ Which counts as two factor authentication?

✅ 4. A PIN and a hard token.
💡 2FA requires two DIFFERENT factor types: something you know (PIN), something you have (hard token). Option 1: both something you have. Option 2: username is not a factor. Option 3: both something you know.

❓ Collections of data a person can use to prove identity?

✅ 4. Credentials.
💡 Credentials are the data used to prove identity. Attributes are characteristics. Certificates are a specific type. Shared secrets are a specific authentication mechanism.

❓ Decisions on who can access certain data are best made by...

✅ 1. Data owner.
💡 The data owner knows the sensitivity and value of their data best. This is the principle behind discretionary access control.

❓ Shared secret usually refers to...

✅ 1. A result of protocols like IKE (Internet Key Exchange) of IPsec.
💡 A shared secret is a value known only to communicating parties, established through key exchange protocols like IKE.

📄 Page 9: Special Basic Protections — CA, Challenge-Response, Digital Signature, Firewall, Separation of Duties, Steganography

❓ One task of a certification authority is...

✅ 4. To publish a list of revoked certificates or provide a service that returns the certificate status.
💡 CA tasks include issuing, signing and managing certificate revocation through CRL or OCSP. CAs do NOT maintain private keys.

❓ Challenge-response is a protocol such that...

✅ 4. A challenges B to respond with something that no one else than A — and possibly B itself — would be able to produce.
💡 Challenge-response sends an unpredictable challenge only the legitimate party can answer using their secret.

❓ A digital signature is made with...

✅ 4. A hash function and an asymmetric cryptoalgorithm.
💡 Digital signature: hash the message then encrypt with private key. Recipient decrypts with public key and compares hashes.

❓ At what phase of a product's life cycle is it most expensive to improve security?

✅ 1. Implementation.
💡 The later in the development lifecycle you fix security the more expensive it is. Implementation is the latest stage among the options.

❓ What does a firewall prevent from functioning in the way they were intended to?

✅ 4. Packets.
💡 A firewall inspects and filters network packets. Ports, IP addresses and protocols are rule attributes but packets are what get blocked.

❓ Which security concept is violated by one person inserting vendors AND paying them?

✅ 3. Separation of duties.
💡 Separation of duties requires critical tasks be divided among different people so no single person can commit fraud alone.

❓ Steganography is...

✅ 2. An art of embedding data into a larger quantity of other data.
💡 Steganography hides a secret message within ordinary data like images or audio. Message is hidden not encrypted. Different from digital watermarking.

📄 Page 10: Repair and Punishment — Backup, Integrity, Privacy, GDPR

❓ Which statement about backups is true?

✅ 2. Removable flash memories are suitable as backup media.
💡 Flash drives are valid backup media. Option 1 wrong — printing IS valid for some data. Option 3 wrong — weekly backups are better than none. Option 4 wrong — incremental copies only copy changes.

❓ Why is off-site backup important?

✅ 4. Prevent the loss of data in the event of a fire.
💡 Off-site backup protects against physical disasters that destroy both primary data and on-site backups simultaneously.

❓ Most concrete concept for repairing broken integrity?

✅ 4. Redundancy.
💡 Redundancy means having extra copies or checksums that allow detection and restoration of corrupted data.

❓ If your personal data is unduly disclosed by an EU service...

✅ 3. The responsible company may have to pay a fine of millions of euros.
💡 Under GDPR, fines up to 20 million euros or 4% of global annual turnover. Company is responsible even if attacker is not caught.

📄 Page 11: More Than GDPR — Data Protection, GDPR, PCI-DSS

❓ The central meaning of data protection is in protecting...

✅ 2. The information that individuals disclose to different data collectors.
💡 Data protection is about protecting personal information that individuals share with organizations.

❓ GDPR defines several roles but NOT the role of data...

✅ 2. Object.
💡 GDPR defines: data subject, data controller, data processor. Data object is a technical database term not a GDPR legal concept.

❓ GDPR mainly covers...

✅ 2. The duties of organizations when they handle data from individuals.
💡 GDPR primarily places obligations on organizations. Not just cross-border exchange. Not about encryption standards.

❓ PCI-DSS is...

✅ 3. Is, as its name says, a standard created by an industry branch.
💡 PCI-DSS was created by major card brands (Visa, Mastercard, Amex, Discover, JCB) not by a government. Industry standard not a law.

📄 Page 12: Anonymity — Anonymity, Privacy, Pseudonym

❓ Anonymity has different degrees depending on...

✅ 3. What data should not be disclosed and to whom.
💡 Anonymity depends on WHAT information should be hidden AND WHO it should be hidden from. Country and age are irrelevant.

❓ Privacy is synonymous to...

✅ 3. None of these.
💡 Privacy is distinct from secrecy, anonymity and privilege. Privacy is the right to control your own personal information.

❓ Pseudonym is...

✅ 4. An artificial name that replaces the real name of a user.
💡 A pseudonym is a false name used instead of a real name. Not a filesystem term. Not inherently part of an attack.

📄 Page 13: IPR — Intellectual Property, Copyright, DRM

❓ Copyright is originally a property of...

✅ 2. The creators of an artefact.
💡 Copyright belongs to the creator by default. Publishers may acquire rights through contracts but do not own copyright originally.

❓ One purpose of DRM is...

✅ 1. To provide access control to digital media for protection of copyrights.
💡 DRM controls how digital content can be accessed, copied and distributed. Option 2 describes anti-circumvention laws. Option 3 wrong — DRM restricts not promotes access.

❓ What is NOT a way to protect intellectual property?

✅ 1. Privacy regulations.
💡 Privacy regulations like GDPR protect personal data not intellectual property. DRM, patents and licensing are all genuine IP protection mechanisms.

📄 Page 14: Social Media — Fake News, Background Checks, Misinformation Responsibility

❓ What can go wrong when including social media links in a job application?

✅ 2. The recruiter follows what is linked to those postings, and finds something unfavourable of you.
💡 Your social media may contain posts or connections that create a negative impression. Following public links is not a privacy offense.

❓ What kind of update is poorly taken care of in social media?

✅ 1. Rectifying of fake news or alternative truths.
💡 Social media spreads content quickly but is very slow to correct misinformation. False information stays up and is rarely retracted.

❓ What kind of update is usually poorly handled?

✅ 1. Links to sites that have published new pages that correct earlier mistakes in their news.
💡 Corrections to previously shared false information are almost never propagated. Original posts remain unchanged.

❓ Who is responsible if you share something legal but untrue that causes harm?

✅ 2. You, without conditions.
💡 If you share false information that causes harm you bear responsibility whether you created it or reshared it.

📄 Page 15: Ethics — Ethics in InfoSec, Anonymity of Browsing, TOR

❓ Ethics is an important topic in information systems...

✅ 1. And some aspects of it extending to the area of information security include treatment of privacy and IPR.
💡 Ethics in information security covers privacy, IPR, responsible disclosure and fair use. Option 2 too narrow. Option 3 wrong — ethics plays a major role. Option 4 wrong — ethical hacking is just one aspect.

❓ What is NOT a topic of ethical considerations?

✅ 3. Wasting your own and others' time by writing in social media long explanations of questionable quality.
💡 This is a matter of quality and courtesy but not a genuine ethical dilemma in information security. Options 1, 2 and 4 all involve real ethical questions.

❓ If you need to browse the internet anonymously...

✅ 2. The TOR network is an option you can seriously consider.
💡 TOR is a legitimate and legal tool routing traffic through multiple encrypted layers. Option 1 wrong — public machines may log activity. Option 3 wrong — private mode only prevents local browser history. Option 4 wrong — TOR and VPNs are legal in Finland.

📄 Page 16: Security Issues on Personal Devices — Cookie, Entropy, Firewall, USB

❓ You get logged in automatically next day without entering credentials — what can you infer?

✅ 4. The site uses a login cookie that persists even if the browser is not running.
💡 A persistent login cookie stored on your computer is sent automatically when you revisit the site. Browsers do not send passwords automatically.

❓ USB stick with 10-digit keyboard, longest passcode is 10 digits — worst case attempts?

✅ 1. 10^10
💡 You chose only 10-digit codes so attacker needs to try all 10-digit combinations. Each position has 10 values (0-9) giving 10^10. Option 2 wrong — attacker only tries 10-digit codes. Option 3 wrong — attacker attacks the passcode not the AES key.

❓ A personal computer firewall is typically...

✅ 2. One process among others.
💡 On a personal computer a firewall is a software process running alongside other processes. Not hardware on the network card. Not an accessory on the power cord.

❓ Which USB connection has the least information security risk?

✅ 3. Charging the new camping shower you just bought from a Chinese web shop.
💡 A simple water pump only needs power and has no data capability. Option 1 has document malware risk. Option 2 risks juice jacking. Option 4 risks compromise despite IT check.

📄 Page 17: Risk Management, Policy — Security Policy, Risk Analysis, Risk Treatments

❓ What control mechanisms can be left out of risk analysis?

✅ 3. Mechanisms that do not control threats to the system under consideration.
💡 Risk analysis should only include controls relevant to the threats being analyzed. Controls addressing threats outside the system scope are irrelevant.

❓ Which does NOT need to be considered in risk analysis?

✅ 1. How much is invested in information security in competing companies.
💡 What competitors spend on security is irrelevant to your own risk analysis. Your analysis is about YOUR assets, YOUR threats and YOUR controls.

❓ Overall risk is reduced if...

✅ 4. The value of an asset is found to be smaller than before.
💡 Risk = Threat x Vulnerability x Asset Value. If an asset is worth less than previously estimated the real risk is genuinely lower.

❓ Alice implements security controls to protect email — this is...

✅ 1. Risk mitigation.
💡 Risk mitigation means taking active steps to reduce risk by implementing controls. The four risk treatments are: mitigation, acceptance, transference and avoidance.

📄 Page 18: Non-disclosure, but Public Keys — NDA, PKI

❓ Usual purpose of a non-disclosure agreement?

✅ 2. To prevent an employee or visitor from revealing sensitive information of an organization.
💡 An NDA is a legal contract binding a person to keep organizational information confidential. Option 1 describes data protection/GDPR. Option 3 describes a classification policy. Option 4 describes an encryption policy.

❓ Basic goal of a public key infrastructure is to...

✅ 2. Bind the public key to the entity to which the key belongs.
💡 PKI creates a trusted link between a public key and the identity of its owner. Option 1 wrong — private and public keys are already mathematically linked. Options 3 and 4 describe structural features not the basic goal.

❓ A certificate for K is generated when...

✅ 1. Someone else uses his private key for a signature that binds K and the identity of B.
💡 A certificate is created when a trusted third party (CA) uses its OWN private key to sign a binding between public key K and the identity of B. The someone else is the CA not B.

📄 Page 19: Password — Entropy, Password Rules

❓ Most important for security question/answer from authentication point of view?

✅ 2. The entropy of the answer.
💡 From an authentication perspective the answer must be unpredictable — high entropy means hard to guess. The question is often visible to attackers so its entropy matters less.

❓ Most important condition for accepting browser password save?

✅ 3. The passwords in your browser are protected from others.
💡 If someone else can access your browser they see all your saved passwords. This is the critical security condition.

❓ First a prohibition you MUST NOT relax, then one you CAN relax?

✅ 1. (ii) and (iii)
💡 You must NEVER tell your password to anyone (ii). But you CAN save your password (iii) in a trusted password manager.

❓ Most dangerous password behavior?

✅ 1. Uses the same password in several places.
💡 Password reuse is most dangerous because one breach compromises ALL accounts sharing that password — credential stuffing attacks.

❓ How many features does the toothbrush analogy represent?

✅ 2. Two.
💡 A toothbrush represents: (iii) personal use only — never share it, and (iv) change it often enough. Does NOT represent entropy (i) or usability (ii).

📄 Page 20: Cookie — Web Browser Cookies

❓ A web browsing cookie cannot break...

✅ 3. Either the integrity or confidentiality of your files.
💡 A cookie is a small piece of data stored in the browser — it cannot directly access or modify files on your computer. It affects your session and web data not your local file system.

❓ In original usage, a cookie is created by...

✅ 4. The web server, which gets it back from the browser without changes.
💡 The web server creates the cookie, sends it to the browser, the browser stores it, and sends it back unchanged with subsequent requests. This allows the server to maintain session state.

❓ Purpose of a web browsing cookie is to transmit to the server information...

✅ 1. That is the same as the server has earlier transmitted to the browser.
💡 The cookie sends back to the server the same information the server originally sent. This is how the server maintains state.

📄 Page 21: CAPTCHA — CAPTCHA, Spam, Email Obfuscation

❓ CAPTCHA is...

✅ 2. Is an example of authentication that does not involve identification.
💡 CAPTCHA proves you are human without knowing WHO you are (no identification). Option 1 wrong — CAPTCHAs can be attacked by human farms and AI. Option 3 wrong — it IS a form of authentication.

❓ Why write email as john.doePOISTA@MINUTtuni.fi?

✅ 1. To avoid spammers harvesting your real address so easily.
💡 This is email address obfuscation. Spambots scan web pages for email patterns. Adding Finnish words POISTA MINUT (Remove me) breaks the standard email pattern so bots cannot harvest it.

📄 Page 22: Crypto Procedures — Cryptography, Primitive, Algorithm, Protocol, Plaintext

❓ Cryptographic primitives are mainly...

✅ 3. Procedures that a cryptology practitioner can view like indivisible blocks.
💡 Primitives are the basic building blocks of cryptographic systems — like hash functions, block ciphers, stream ciphers. They are treated as indivisible black boxes combined to build larger protocols.

❓ Which is mainly a task performed using a cryptographic protocol?

✅ 4. Agreeing on an encryption key.
💡 A cryptographic protocol involves multiple parties exchanging messages to achieve a goal. Key agreement (like Diffie-Hellman) requires communication between parties.

❓ Crypto algorithms are either symmetric, asymmetric or signature — true or false?

✅ 1. False because the list lacks algorithms that do not use a key.
💡 Hash functions are cryptographic algorithms but use no key. The statement is false because it omits keyless algorithms.

❓ Which is NOT covered by the concept of a cryptographic algorithm?

✅ 2. Challenge-response method.
💡 Challenge-response is a PROTOCOL not an algorithm — it involves multiple steps and parties exchanging messages.

❓ Implementation of ____ containing all software, protocols, algorithms and keys is called ____.

✅ 2. Cryptography, a cryptosystem.
💡 Cryptography is the practice/science. A cryptosystem is a complete implementation containing all necessary components.

❓ The term plaintext...

✅ 4. Is a synonym for cleartext and the concept is used in cryptography.
💡 Plaintext in cryptography means unencrypted data. It is the data BEFORE encryption. Its counterpart is ciphertext.

📄 Page 23: Keys and Chunks or Streams — Encryption, Key Length, Block Cipher, Stream Cipher

❓ Two classes of symmetric cryptographic methods — blank and blank ciphers.

✅ 2. Block, stream.
💡 Block ciphers process plaintext in fixed-size blocks (e.g. 128 bits at a time). Stream ciphers process data bit by bit.

❓ Encryption requires a key that...

✅ 4. Can even be computed from a human-memorable password if the data is not to be sent anywhere but protected in local storage.
💡 This describes password-based encryption. Key derivation functions (KDF) convert passwords into encryption keys.

❓ Doubling symmetric key length — brute force time T becomes...

✅ 4. T squared.
💡 If original key is n bits, brute force tries 2^n = T keys. Doubling to 2n bits means 2^(2n) = (2^n)^2 = T squared. Doubling key LENGTH squares the brute force effort.

❓ Stream ciphers — the other class of symmetric methods is...

✅ 4. Block algorithms.
💡 The two classes of symmetric cryptographic methods are stream ciphers and block ciphers.

📄 Page 24: One-time Pad, DES, AES — Encryption, AES, One-time Pad

❓ Most popular symmetric cryptosystem currently?

✅ 1. AES.
💡 AES (Advanced Encryption Standard) replaced DES and is now the most widely used symmetric encryption algorithm worldwide. DES is obsolete. RSA is asymmetric. TLS is a protocol.

❓ DES was for a long time the state-of-the-art method to...

✅ 3. Produce confidentiality of communications.
💡 DES was the standard symmetric encryption algorithm. Eventually broken due to its short 56-bit key length.

❓ What is a one-time pad?

✅ 2. A single-use key together with the method of using it to encrypt and decrypt a message.
💡 A one-time pad is a theoretically unbreakable encryption method where the key is truly random, at least as long as the message, and used only ONCE. Provides perfect secrecy.

❓ AES is a...

✅ 2. Block cipher.
💡 AES is a symmetric block cipher processing data in 128-bit blocks using keys of 128, 192 or 256 bits. Option 1 wrong — AES is a block cipher not a stream cipher.

📄 Page 25: Public Key — Private Key, Public Key, RSA

❓ Public key for _____ is like a physical key that can SHUT (lock) a padlock?

✅ 1. Encryption.
💡 In asymmetric encryption anyone can use the public key to lock (encrypt) data, but only the private key holder can unlock (decrypt) it.

❓ Public key for _____ is like a physical key that can OPEN a padlock locked with a private key?

✅ 3. Verifying signatures.
💡 In digital signatures the sender signs with their private key and anyone can verify using the public key.

❓ Which does NOT apply about RSA? (first set)

✅ 1. If the RSA decryption exponent is d, then the RSA decryption program has to do d-1 modular multiplications.
💡 FALSE. RSA uses square-and-multiply fast exponentiation requiring approximately log(d) multiplications, NOT d-1.

❓ Which does NOT apply about RSA? (second set)

✅ 3. An RSA signature cannot be made unless you know the two prime numbers that make up the public modulus n.
💡 FALSE. RSA signing only requires private exponent d and modulus n — NOT the prime factors p and q.

📄 Page 26: Hash — Checksum, Hash, Keyed Hash

❓ Typical of checksums for preventing typing errors in digit strings?

✅ 2. They are calculated from all the characters.
💡 A checksum must cover all characters in the string to detect errors anywhere in the input.

❓ Cryptographic hash represents the entire message in the sense that...

✅ 2. It is very unlikely to find any other message with the same hash.
💡 This describes collision resistance. Option 3 is opposite — avalanche effect means LARGE hash changes from small message changes.

❓ How does a cryptographic hash protect a message?

✅ 3. The message cannot be changed without breaking the correspondence with the value.
💡 Any change to the message produces a completely different hash, revealing tampering. This is integrity protection.

❓ Which does NOT apply to keyed cryptographic hashes?

✅ 2. They are also called Cyclic Redundancy Checks.
💡 CRCs are non-cryptographic error detection checksums — completely different from keyed cryptographic hashes (MACs).

❓ Bit strings of length 100 giving H(m) starting with 40 zeros?

✅ 4. 2^60.
💡 There are 2^100 possible 100-bit strings. Probability of hash starting with 40 zeros = 1/2^40. Expected count = 2^100 / 2^40 = 2^60.

❓ Bit strings of length 40 giving H(m) starting with 100 zeros?

✅ 1. Most likely none.
💡 There are only 2^40 possible 40-bit strings. Probability = 1/2^100. Expected count = 2^40 / 2^100 = essentially zero.

📄 Page 27: Most Common Protocols — KEX, SSH, TLS, IPsec

❓ IPsec...

✅ 1. Provides mechanisms for authentication and encryption.
💡 IPsec is a suite of protocols providing both authentication and encryption at the IP layer. Used in VPNs. Works with both IPv4 and IPv6.

❓ What does key exchange mean in cryptographic protocols?

✅ 4. A symmetric key is agreed upon.
💡 Key exchange means two parties agreeing on a shared symmetric key (like Diffie-Hellman) without it ever being transmitted directly.

❓ Protocol and program for remote command line use?

✅ 2. SSH.
💡 SSH (Secure Shell) is the standard protocol for securely connecting to a remote computer command line.

❓ The T in TLS stands for...

✅ 3. Transport.
💡 TLS stands for Transport Layer Security. It replaced SSL and secures communications at the transport layer.

📄 Page 28: Nature and Devices — Fire, Smart Card

❓ What is true about fires?

✅ 3. It is not so much the computers that catch fire, but the batteries and their charging are causing risks especially in mobile devices.
💡 Lithium-ion batteries in mobile devices are a genuine fire risk. IT systems have special fire suppression requirements (gas not water).

❓ Why cannot digital cash bits be extracted and copied from a smart card?

✅ 4. The card processor only transacts with processors of the right kind and even then spending happens inside the card.
💡 Smart cards have a secure processor that handles all transactions internally. Bits never leave the card in a usable form.

📄 Page 29: Filtering — DMZ, Packet Filter

❓ A DMZ means a part of an organization network that...

✅ 1. Resides between two filtering devices.
💡 A DMZ sits between two firewalls — one facing the internet and one facing the internal network. It hosts public-facing servers.

❓ Which should mainly be able to filter packets like a dedicated firewall?

✅ 3. Router.
💡 Routers operate at the network layer (Layer 3) and can perform packet filtering based on IP addresses and port numbers.

❓ What can a packet filter do: anti-virus, delete oversized attachments, encrypt/decrypt?

✅ 1. None of these.
💡 A packet filter operates at the network layer inspecting only packet headers. It cannot inspect content for viruses, read email attachments, or encrypt/decrypt traffic.

❓ How does a packet filter react to TCP and UDP port numbers?

✅ 4. It passes through the packet if and only if those numbers are equal.
💡 A packet filter uses port numbers to decide — it passes packets whose port numbers match the configured allowed ports in the rules.

❓ Tasks of a packet filter may include...

✅ 4. Writing logs.
💡 Packet filters CAN log traffic. They CANNOT perform user authentication, anti-virus scanning or spam filtering.

📄 Page 30: Secure Connection — IPsec, TLS, VPN, Remote Use

❓ IPsec adds to each data packet a field with...

✅ 4. A hash calculated from the packet and a symmetric key.
💡 IPsec Authentication Header (AH) adds an HMAC — a hash calculated from packet contents using a shared symmetric key. Provides integrity and authentication.

❓ IPsec encrypts at a layer that is... compared to TLS?

✅ 3. Lower than in TLS.
💡 IPsec operates at Layer 3 (Network/IP layer). TLS operates between Layer 4 (Transport) and the Application layer. Layer 3 is lower than Layer 4.

❓ If user fails to authenticate target machine, which threat is most serious?

✅ 1. Give the user authentication information to an attacker.
💡 If the user connects to a fake server without authenticating it, the user enters credentials which the attacker captures. This compromises all services where those credentials are used.

❓ Because TLS works below the application layer, it...

✅ 1. Does not know what kind of data it protects.
💡 TLS encrypts whatever the application sends without knowing whether it is HTTP, email, FTP or any other type. It is transparent to the application protocol.

❓ What is more important in a VPN for remote worker security?

✅ 2. Encryption.
💡 The fundamental security requirement of a VPN for remote workers is encryption — protecting data as it travels over the public internet.

📄 Page 31: Mobile Authentication — Cellular, WiFi, WLAN, WPA

❓ What mechanism authenticates the SIM card in cellular networks?

✅ 2. Challenge-response method, with a symmetric key system.
💡 GSM and cellular networks use challenge-response authentication with a shared symmetric key stored on the SIM card. No public key system involved.

❓ How do 2G, 3G, 4G systems authenticate the subscriber?

✅ 4. By applying a cryptographic hash function to a shared secret.
💡 Mobile networks apply a hash/HMAC to a shared secret combined with a random challenge. The shared secret is NEVER transmitted. Cellular auth uses symmetric not PKI.

❓ Relation between WiFi, WLAN, VLAN and WPA?

✅ 1. WPA is the protection method for a WLAN, which uses the WiFi protocol, and can be one of the VLANs in an organization network.
💡 WiFi is the wireless networking protocol. WLAN is the network type. WPA is the security protocol. VLAN is a separate network segmentation concept.

❓ WPA protects which wireless technologies: NFC, ZigBee, Bluetooth, WiFi?

✅ 2. (iv) WiFi only.
💡 WPA was specifically designed to protect WiFi/WLAN communications only. NFC, ZigBee and Bluetooth all have their own separate security mechanisms.

📄 Page 32: Start Running Programs — Login, Single Sign-on, Operating System, Sandbox

❓ What is NOT used as a term for logging into a system?

✅ 2. Pass in.
💡 Log in, sign in and log on are all common synonymous terms. Pass in is not a standard term used for this purpose.

❓ Login is exceptional because...

✅ 2. It need not know whether it is launched by a legitimate user or an attacker.
💡 Login must handle authentication requests from anyone. It processes whatever credentials are presented and grants or denies access.

❓ Most essential protection for your computing is provided by...

✅ 2. The operating system.
💡 The operating system is the foundation of all security — it controls access to hardware, manages memory isolation, enforces permissions. Everything else depends on and runs within the OS.

❓ Sandboxing is a security mechanism where...

✅ 1. New untrusted software is run in a disconnected test system to check it for malicious behaviour.
💡 Sandboxing isolates untrusted code in a restricted environment. The sandbox limits what resources and system calls the code can access.

❓ Single sign-on...

✅ 1. Can provide access to multiple resources, but it is only valid for a limited session or duration.
💡 SSO allows a user to authenticate once and gain access to multiple systems. The session is time-limited for security.

❓ Single sign-on is characterized by...

✅ 4. Security is improved compared to a user having the same password for multiple services.
💡 With SSO users only need one strong password rather than reusing the same password across multiple services.

📄 Page 33: A Spectrum of Storage Issues — Database, Overwriting, Password Salt

❓ Specific security requirements for databases concern more than other options...

✅ 2. The data that your (future) employer has about its employees.
💡 Employee databases contain structured sensitive personal data with complex dependencies. A folder of papers is not a database. A photo album has minimal structured dependencies.

❓ Why write zeros and ones over and over again on a file on disk?

✅ 4. So that the earlier contents of the file could not be read at all.
💡 Overwriting data multiple times is a secure deletion technique. Magnetic storage retains traces of previous data even after deletion. Multiple overwrites make recovery extremely difficult.

❓ Password salt is...

✅ 4. The random number stored without encryption together with each hashed password.
💡 A salt is a random value added to each password before hashing, stored alongside the hashed password in plaintext. It ensures two users with the same password have different hashes, defeating rainbow table attacks.
